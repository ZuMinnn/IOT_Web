Mô tả:

Bước 1: Thiết bị đọc dữ liệu từ cảm biến
Hardware đọc dữ liệu từ 2 cảm biến
* Cảm biến nhiệt độ/ độ ẩm.
* Cảm biến ánh sáng.
Bước 2: Thiết bị publish dữ liệu cảm biến
* Tác nhân: Hardware
* Giao thức: MQTT
* Hành động: Gửi dữ liệu đo được lên MQTT Broker
Topic sử dụng: iot/sensor/env
Bước 3: Backend API subcribe và tiếp nhận dữ liệu
* Tác nhân: Backend API
* Cơ chế: MQTT Subcriber
Backend thực hiện subcribe topic khi có message mới callback onMessage() được kích hoạt, backend parse payload và tách dữ liệu từng cảm biến
Bước 4: Backend API xử lý và lưu dữ liệu vào Database
Sử dụng Internal API  để lưu dữ liệu vào bảng SensorData
Bước 5: Database phản hồi kết quả lưu dữ liệu

Phần III: Thiết kế cơ sở dữ liệu

1. Sơ đồ cơ sở dữ liệu
Entity Relationship Diagram (ERD)

    
    
    

CộtKiểuÝ nghĩa    IDintegerKhóa chính, định danh duy nhất cho mỗi thiết bị    namevarcharTên thiết bị     createAtdateNgày tạo thiết bị trong hệ thống    
CộtKiểuÝ nghĩa    IDintegerKhóa chính của bản ghi hành động    deviceIDintegerKhóa ngoại → Device.ID    actionvarcharHành động được gửi (ON / OFF / SET_VALUE...)    statusvarcharTrạng thái sau khi thực hiện (SUCCESS / FAIL)    runningintegerTrạng thái thực thi (0: chưa chạy, 1: đang chạy, 2: xong)    datedateThời điểm gửi lệnh    createAtdateThời điểm lưu bản ghi    








CộtKiểuÝ nghĩa    IDintegerKhóa chính    namevarcharTên cảm biến (VD: Nhiệt độ, Độ ẩm, Ánh sáng)    createAtdateNgày thêm cảm biến    

CộtKiểuÝ nghĩa    IDintegerKhóa chính    SensorIDintegerKhóa ngoại → Sensor.ID    valuedoubleGiá trị đo được     datedateThời điểm đo
